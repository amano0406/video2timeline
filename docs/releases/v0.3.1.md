# TimelineForVideo v0.3.1 Tech Preview

Local-first video-to-markdown packaging tool for LLM workflows.
This is a desktop-style local tool, not a hosted SaaS product.

## Baseline Support

- Windows
- Docker Desktop
- CPU mode

## Optional Features

- GPU mode: NVIDIA + Docker GPU access only, best-effort
- Speaker diarization: optional, requires both a Hugging Face token and gated approval for `pyannote/speaker-diarization-community-1`

## Download

- Windows: `TimelineForVideo-windows-local.zip`
- macOS: source-based, experimental path

## What's New

- `failed` jobs can now export successful timelines as ZIP files when at least one timeline completed
- ZIP exports now include `FAILURE_REPORT.md` and `logs/worker.log` when warnings or failed items exist
- worker reliability improved for concurrent status writes and GPU failures now fall back to CPU more safely
- user-facing navigation and wording now use `job` consistently, with `/jobs/...` as the canonical URL path
- the Jobs list now uses the same human-friendly remaining-time style as Job Details
- file sizes are now shown in compact units such as `5.2 GB` or `842 MB` instead of long raw counts

## Known Limitations

- first run downloads models and takes time
- macOS is experimental
- GPU is not the baseline support path
- GUI is the primary supported path
- Docker Desktop is required
