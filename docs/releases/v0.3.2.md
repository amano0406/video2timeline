# TimelineForVideo v0.3.2 Tech Preview

Local-first video-to-markdown packaging tool for LLM workflows.
This is a desktop-style local tool, not a hosted SaaS product.

## Baseline Support

- Windows
- Docker Desktop
- CPU mode

## Optional Features

- GPU mode: NVIDIA + Docker GPU access only, best-effort
- Speaker diarization: optional, requires both a Hugging Face token and gated approval for `pyannote/speaker-diarization-community-1`

## Download

- Windows: `TimelineForVideo-windows-local.zip`
- macOS: source-based, experimental path

## What's New

- duplicate uploads can now reuse existing timelines and ZIP exports instead of ending in empty job details
- stale duplicate catalog entries now fall back to fresh processing instead of silently skipping missing artifacts
- Job Details now supports rerunning with either the original job settings or the current app settings
- the Jobs page active panel now prefers a running job over a newer pending job

## Known Limitations

- first run downloads models and takes time
- macOS is experimental
- GPU is not the baseline support path
- GUI is the primary supported path
- Docker Desktop is required

## Verification

- `dotnet build web/TimelineForVideo.Web.csproj`
- `PYTHONPATH=worker/src python3 -m unittest discover worker/tests`
- `dotnet test tests/TimelineForVideo.E2E/TimelineForVideo.E2E.csproj`
- live `/jobs` check confirmed after deploy
